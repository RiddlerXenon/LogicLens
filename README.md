# LogicLens

> Автоматическое обнаружение и объяснение логических ошибок в тексте.

**Tech Stack:** `xlm-roberta-base` • `Qwen3-8B` • `FastAPI` • `JavaScript`

---

## О проекте

LogicLens — система анализа аргументации, которая автоматически определяет логические ошибки в тексте и объясняет, почему аргумент является некорректным.

Проект объединяет:

- классификатор логических ошибок на основе **XLM-RoBERTa**
- генерацию объяснений при помощи **Qwen3-8B**
- REST API на **FastAPI**
- веб-интерфейс на чистом HTML/CSS/JavaScript

---

## Возможности

### Анализ текста

Пользователь вводит произвольный текст, после чего система:

- определяет тип логической ошибки;
- показывает вероятность классификации;
- отображает Top-3 наиболее вероятных классов;
- генерирует подробное объяснение ошибки;
- предлагает рекомендации по улучшению аргумента.

### Режим дебатов

Два пользователя вводят свои аргументы.

Система:

- анализирует оба текста независимо;
- вычисляет оценку логичности каждого аргумента;
- формирует краткий отзыв каждому участнику;
- генерирует итоговый вердикт;
- определяет победителя или объявляет ничью.

Система состоит из трёх основных компонентов:

- **Классификатор** — определяет тип логической ошибки.
- **LLM** — объясняет найденную ошибку.
- **FastAPI** — объединяет модели в единый сервис.

---

# Классификатор

Основан на модели **xlm-roberta-base**.

Обучение выполнено на переведённом датасете **LOGIC (Jin et al., EMNLP 2022)**.

### Поддерживаемые классы

| Класс | Описание |
|-------|----------|
| ad hominem | Атака на личность |
| ad populum | Апелляция к большинству |
| appeal to emotion | Апелляция к эмоциям |
| circular reasoning | Круговое доказательство |
| equivocation | Подмена понятий |
| fallacy of credibility | Апелляция к авторитету |
| fallacy of extension | Приписывание чужих слов |
| fallacy of logic | Нарушение формальной логики |
| fallacy of relevance | Нерелевантный аргумент |
| false causality | Ложная причинность |
| false dilemma | Ложная дилемма |
| faulty generalization | Поспешное обобщение |
| intentional | Намеренное введение в заблуждение |

### Качество модели

| Язык | F1-macro |
|-------|----------|
| Английский | **0.587** |
| Русский | **0.506** |

# Генерация объяснений

Используется локально запущенная модель **Qwen3-8B**.

Она получает:

- исходный текст;
- тип найденной ошибки;
- описание класса.

После чего генерирует:

- объяснение ошибки;
- причину некорректности аргумента;
- пример похожей ситуации;
- рекомендации по исправлению.

# API

## Проверка состояния

```
GET /health
```

Ответ

```json
{
  "status": "ok",
  "classifier": true,
  "llm": true,
  "device": "cuda"
}
```

---

## Анализ текста

```
POST /analyze
```

Возвращает:

- найденную логическую ошибку;
- вероятность;
- Top-3 классов;
- объяснение LLM.

---

## Анализ дебатов

```
POST /debate
```

Возвращает:

- результаты анализа обоих участников;
- оценки логичности;
- комментарии модели;
- итоговый вердикт.

# Запуск

## Требования

- Python 3.10+
- CUDA GPU (рекомендуется ≥16 GB VRAM)
- обученный классификатор
- локальная модель Qwen3-8B

---

## Установка

```bash
pip install fastapi uvicorn transformers torch accelerate pydantic
```

---

## Запуск API

```bash
uvicorn main:app --host 0.0.0.0 --port 8000 --workers 1
```

> Используйте `--workers 1`, поскольку одновременно загружать несколько экземпляров Qwen3-8B не рекомендуется.

---

## Запуск фронтенда

```bash
python -m http.server 3000
```

После запуска откройте

```
http://localhost:3000/index.html
```

# Ограничения

- отсутствует класс **No Fallacy**;
- качество модели на русском ниже английской версии;
- редкие классы (например, *equivocation*) определяются значительно хуже;
- время работы режима дебатов примерно в 3 раза больше режима анализа.
